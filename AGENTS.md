# AGENTS.md

Repo-specific guidance for OpenCode sessions working on TaskMaster (PHP 8.3
to-do manager with web + CLI faces). See `README.md` for user-facing setup.

## Commands

- QA (lint dry-run): `composer qa` — runs `php-linter`, `php-cs-fixer
  --dry-run`, `phpstan analyse` (level 8), `rector --dry-run`.
- Auto-fix: `composer lint:fix` — runs `php-cs-fixer fix` then `rector process`.
- Dev web server: `php -S localhost:8000 -t public/`.
- CLI: `./bin/task.php process|add|search` (see README for flags).
- Markdown lint/format: `npm run lint` / `npm run format`.

### Gotchas

- **`composer test`, `test:unit`, `test:integration` are currently broken.**
  There is no `phpunit.xml`, no `tests/` directory, and `phpunit` is not in
  `require-dev`. Do not assume tests run; verify before relying on them.
- **`docs/schema.sql` referenced in README does not exist** (`docs/` is empty).
  The DB schema's source of truth is missing from the repo; do not invent it.
- Lint/static-analysis cover only `bin/` and `src/`. `public/index.php`,
  `bootstrap.php`, and `templates/` are NOT checked — review them by hand.
- No CI is actually configured (no `.github/`, no `.gitlab-ci.yml`); the GitLab
  snippet in README is aspirational.
- `bootstrap.php` hard-fails without `config/config.ini` (copy from
  `config/config.ini.sample` and fill DB creds) and a reachable MySQL/MariaDB.
  Both entrypoints (`public/index.php`, `bin/task.php`) depend on it.

## Architecture

- Two entrypoints: `public/index.php` (web front controller) and
  `bin/task.php` (CLI). Both `require` `bootstrap.php`, which returns
  `['pdo' => $pdo, 'twig' => $twig]`.
- Web routing is a simple `?page=` match (`tasks` default, `recurring`).
  Mutations go via `?ajax=` (optionally `?type=recurring`) and return JSON;
  see `public/index.php` for the action allow-list.
- Layers: `src/Controller/` (request handling) -> `src/Persistence/` (PDO
  repositories) -> `src/Task.php` / `src/Frequency.php` (domain).
  `src/TaskProcessor.php` materializes recurring tasks into one-time tasks.
- Recurring tasks are only generated by running `bin/task.php process`
  (intended to be cron'd every minute). The web UI does not process them.
- Twig templates live in `templates/` (cache disabled, debug on); static
  assets in `public/css` and `public/js`.

## Conventions

- PSR-12 + `declare(strict_types=1)` (enforced by rector); 4-space indent.
- QA order before commit: `composer qa` (optionally `composer lint:fix`).
- Markdown: prettier + markdownlint enforce 100-col wrap (`MD013`), ATX
  headings, dash lists, 2-space list indent. Keep `.md` edits within these.
- `var/`, `vendor/`, `node_modules/`, `config.ini` are gitignored.
